Colours

Two categories of colours are used. Functional colours specify the basic colour palette. Descriptive colours are colours used in a specific context.

A functional colour must always alias a descriptive colour. For example, the brand-green functional colour aliases the green-base descriptive colour.

:root { --colour-brand-green: var(--colour-green-base); }

If a functional colour is used in only a single component, it should be defined in that component's variables block. If a functional colour is used for more than one component, it should be defined globally in colours.css.

Typographic enhancements

typogr.js "provides a set of functions which automatically apply various transformations to plain text in order to yield typographically-improved HTML".

All HTML should be run through typogr at build.

The following Typogr options are enabled:

widont

Replaces the space between the last two words in a string with &nbsp; to avoid a final line of text with only one word.

smartypants

Fixes quotes ( " and ' '"), dashes (“--” and “---”) and ellipses (“...”).

caps

Wraps MULTIPLE CAPITAL LETTERS in <span class="caps"></span> so they can be styled.

Button

The .Button component defines button styles for <button> and <a> elements.

When using a <button> element, always specify a type. When using a <a> element, always add role="button".

Default state

Buttons without modifier classes default to the standard size and blue colour and should be used for the majority of actions.

Disabled state

Set by adding a disabled attribute. The .is-disabled class can be added for clarity.

For <a> elements, you should also remove the href attribute and prevent JavaScript event handlers from firing.

Pressed state

The .is-pressed class is used here to demonstrate this state, but should never be used.

Large modifier

Large buttons should be used for primary save actions.

Small modifier

Small buttons should be used for secondary save actions.

Block modifier

The block modifier should be used to make a button full-width. It adds width: 100%;, changes the display from inline-block to block, and centers the button text.

Action modifiers

Action modifiers help users understand the consequences of different types of action.

Positive

Positive buttons are green. They indicate a successful or positive action.

Warning

Warning buttons are orange. They indicate that care should be taken with this action.

Danger

Danger buttons are red. They indicate a potentially destructive action.

Controls

The Controls components are groupings of form elements.

All form elements used within a .Control must be associated with a label element.

TextControl

The .TextControl component groups simple, text-based fields that share a set of common states. Status and modifier classes can be applied to any field type and can be stacked, e.g.

All input and textarea elements within a .TextControl component must have a name attribute that matches the type of data expected (e.g. <input type="email" name="email">.

The placeholder attribute should be used where this will aid usability but should not be used in place of a label element.

The autocomplete="off" attribute should not be used for password fields, but should be used for other sensitive data types (e.g. credit card information).

The autofocus attribute should only be used once per page, and only on the first field on a composed page that has a single, simple form (e.g. Login).

Currency

The .Currency component must be used to render all representations of a currency. It should not be used for any other type of number.

The <data> element must be used as the root element for this component and the value attribute must be populated with a number fixed to two decimal places.

The rendered currency should follow the Style.ONS guide.

DataActionTable

The .DataActionTable component should be used when rows of tabular data are displayed with associated actions. It currently supports the following column types:

  • Utility column, which should always be centre-aligned and fixed width e.g. warnings count, campaign status, row-specific tools
  • Row name and associated link to detail view
  • Dates, with optional times
  • Percentage data
  • Fractional data e.g. "$10 ⁄ $100"
  • Numeric data, which should always be right-aligned and use tabular figures

DateTime

The .DateTime component must be used to render all representations of dates and times.

The <time> element must be used as the root element for this component and the datetime attribute must be populated with a valid date-time.

The rendered date and time should be generated from the date-time attribute and should follow the Style.ONS guide.

FullPage

.FullPage is a wrapper component intended for use on single-purpose pages. It fills the whole viewport and sets a scaling background image. Any component contained within the .FullPage-body class will be centered.

It can optionally contain a single instance of the .Footer component, which should be contained in the .FullPage-footer descendent.

RawHTML

The .RawHTML component is used to wrap sections of plain, class-free HTML that require default typographical styles and margins. Typographic elements used outside the context of .RawHTML do not have any margins attached.
