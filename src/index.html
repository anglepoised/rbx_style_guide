Typographic enhancements

typogr.js "provides a set of functions which automatically apply various transformations to plain text in order to yield typographically- improved HTML".

All HTML should be run through typogr at build.

The following Typogr options are enabled:

widont

Replaces the space between the last two words in a string with &nbsp; to avoid a final line of text with only one word.

smartypants

Fixes quotes ( " and ' '"), dashes (“--” and “---”) and ellipses (“...”).

caps

Wraps MULTIPLE CAPITAL LETTERS in <span class="caps"></span> so they can be styled.

Block elements

Typographic elements do not come with any margins attached to them in order to make them more flexible in where they can be used.

Button

Standard button styles can be applied to <button>, <a> and <input type="submit"> elements. 

<button class="Button" type="button">
    Button
</button>

<a class="Button" href="{{url}}">
    Link
</a>

<input class="Button" type="submit" value="Submit">

Link

Combinations of states can be applied by stacking class modifiers. For example: 

<button class="Button Button--large Button--orange is-disabled" disabled>
    Large Orange Disabled Button
</button>

Gives:

Core states

Default

Buttons without modifier classes default to the standard size and blue colour and should be used for the majority of actions.

Disabled

Set by adding a disabled attribute. The .is-disabled class can be added for clarity.

For <a> elements, you should also remove the href attribute and prevent JavaScript event handlers from firing. 

<button class="Button is-disabled" disabled>
    Disabled Button
</button>

Pressed

The .is-pressed class is used here to demonstrate this state, but should never be used in raw HTML. 

<button class="Button is-pressed" type="button">
    Pressed Button
</button>

Size variants

Large

Large buttons should be used for primary save actions. 

<button class="Button Button--large" type="button">
    Large Button
</button>

Colour variants

Colour variants are used on buttons to distinguish between different types of action.

Green

Green buttons indicate a successful or positive action. 

<button class="Button Button--green" type="button">
    Green Button
</button>

Orange

Orange buttons indicate that caution should be taken with this action. 

<button class="Button Button--orange" type="button">
    Orange Button
</button>

Red

Red buttons indicate a dangerous or potentially negative action. 

<button class="Button Button--red" type="button">
    Red Button
</button>

Icon variants

Icon button styles can be applied to <button> and <a> elements but not <input type="submit"> elements.

Icons can be used to provide additional context for buttons. They and accept the same class modifiers as standard buttons. For example: 

<button class="Button Button--icon Button--large Button--orange is-disabled" disabled type="button">
    <span class="Icon Icon--white-alabaster-8-plus"></span>
    Large Orange Disabled Icon Button
</button>

Gives:

All button icons should be taken from the white-alabaster set.

Default

Default icon buttons must use 8px x 8px icons.

Large variant

Large variant icon buttons must use 12px x 12px icons.

Hidden text variant

Visible text on icon buttons is optional. If not used, button text should be enclosed between the icon <span> element rather than alongside, and a title attribute should be added. For example: 

<button class="Button Button--icon" title="Button Action" type="button">
    <span class="Icon Icon--white-alabaster-8-plus">Button Action</span>
</button>
